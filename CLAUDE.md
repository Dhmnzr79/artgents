# Правила проекта — artgents

Чистый HTML + CSS + vanilla JS. Без фреймворков, без препроцессоров.

## HTML

- Только семантика там, где уместно: `header`, `main`, `nav`, `section`, `article`, `button`, `footer`.
- Все кликабельные действия — `<button type="button|submit|reset">`, не `div`/`span` как кнопка.
- У каждой кнопки явный `type` (избегать неявного `submit` вне формы).
- Без inline-обработчиков: `onclick`, `oninput`, `onchange` — слушатели только в JS-модуле.
- Не плодить лишних wrapper-ов и не раздувать DOM без причины.

## CSS

- Без `style=""` и без `!important`.
- Без стилей по `#id`; селекторы — плоские, не глубже 3 уровней вложенности.
- Именование — BEM: `.block__element--modifier`.
- Цвета, радиусы, тени, отступы, брейкпоинты — только через CSS custom properties в `:root`.
- Без магических чисел — только токены.
- Media queries — mobile-first (`min-width`), одним блоком в конце файла/компонента, не размазаны по файлу.
- Один компонент — один понятный блок стилей; не копировать одинаковые куски в разные файлы.

## JS

- Без `fetch` и бизнес-логики внутри UI-компонентов — отдельный модуль `api.js` / сервис.
- Без размазанного `document.querySelector` по проекту — привязка к DOM через контейнер или малый модуль инициализации.
- Без глобальных мутабельных переменных.
- Без `console.log` в финальном коде (только через явный debug-флаг).
- Без silent `catch` — ошибки обрабатываются явно.
- Без закомментированного и мёртвого кода.

## Структура файлов

```
/
├── index.html
├── css/
│   ├── tokens.css       # все CSS custom properties
│   ├── base.css         # reset + типографика
│   └── components/      # один файл — один компонент
├── js/
│   ├── api.js           # все fetch-запросы
│   ├── main.js          # точка входа, инициализация
│   └── components/      # UI-модули
└── assets/
    └── images/
```

## Нейминг

- Понятные имена классов, файлов, модулей — по BEM.
- Запрещены: `temp`, `block2`, `wrapNew`, `finalFinal`, `new_new` и т.п.
- API URL, длинные тексты, magic numbers — в константах/конфиге, не хардкод по всему коду.

## Запрещено

| Нельзя | Вместо этого |
|--------|--------------|
| `style=""` | CSS-классы / токены |
| `!important` | специфичность и порядок слоёв |
| `div`/`span` как кнопка | `<button type="…">` |
| inline JS в HTML | модуль + `addEventListener` |
| CSS глубже 3 уровней | плоские селекторы |
| media queries по всему файлу | блок внизу компонента |
| дубли цветов/теней/отступов | токены в `:root` |
| magic numbers | именованные CSS-переменные |
| `console.log` в финале | убрать или debug-флаг |
| закомментированный / мёртвый код | удалить, история в git |
| `fetch` в UI-компоненте | `api.js` / сервис |
| бизнес-логика в UI | данные снаружи, UI только отображает |
| гигантские смешанные файлы | разбить по ответственности |
